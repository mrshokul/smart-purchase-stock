# 📄 SRS (ฉบับย่อ) — ระบบคำนวณอัจฉริยะเพื่อการจัดซื้อและขจัดสินค้าค้างสต็อก

**ประเภทร้าน:** เครื่องเขียน / เครื่องใช้สำนักงาน  |  **ภาษา/สกุลเงิน:** ไทย / บาท (THB)
**รูปแบบระบบ:** เว็บแอปไฟล์เดียว (เปิดในเบราว์เซอร์บน Windows) — ไม่เก็บข้อมูล อัปโหลดใหม่ทุกครั้ง

---

## 1. ภาพรวมระบบ

ระบบช่วยผู้บริหารร้านตัดสินใจ **"สั่งซื้ออะไร-เท่าไร"** และ **"เลิกนำเข้าอะไร (Dead Stock)"** โดยวิเคราะห์จากไฟล์ Excel ที่ export จากระบบขาย (POSPOS) ผู้ใช้อัปโหลดไฟล์เข้ามา ระบบประมวลผลในเครื่องทันที แล้วแสดงผลเป็นการ์ดสรุป + ตาราง + แจ้งเตือน โดย **ไม่เก็บข้อมูลไว้** (เปิดใหม่ต้องอัปใหม่)

หัวใจการคำนวณ: **ใช้ข้อมูลอดีตแบบตามฤดูกาล** — พยากรณ์ความต้องการเดือนเป้าหมายจากยอดขายเดือนเดียวกันย้อนหลัง 3 ปี

---

## 2. ผู้ใช้

| กลุ่มผู้ใช้ | ทำอะไรได้ | เพื่ออะไร |
|---|---|---|
| **ผู้บริหาร** (ผู้ใช้กลุ่มเดียว) | อัปโหลดไฟล์เอง, ตั้งค่า (% เติบโต, เดือนเป้าหมาย, ช่วงเวลา), ดูรายงาน/ตาราง, Export Excel | ตัดสินใจสั่งซื้อ + ขจัดของค้างสต็อก + ดูของขายดี |

ไม่มีระบบล็อกอิน / หลายบทบาท — ผู้บริหารเป็นทั้งคนนำเข้าไฟล์และคนดูผล

## 3. ฟีเจอร์ "ต้องมี" (v1)

| # | ฟีเจอร์ | รายละเอียด / กติกา |
|---|---|---|
| M1 | **อัปโหลดหลายไฟล์** (ไม่จำกัดจำนวน) | แยกประเภทไฟล์อัตโนมัติจากชื่อไฟล์ (ยอดขายรายเดือน / สต็อก / PO / ปฏิทินเทศกาล) และอ่านเดือน-ปีจากชื่อไฟล์ (`YYYY_MM`) |
| M2 | **ตารางแนะนำสั่งซื้อ + จำนวน** | สูตร: `ceil( (ค่าเฉลี่ยยอดขายเดือนเป้าหมาย ย้อนหลัง 3 ปี × (1+%เติบโต)) − สต็อกคงเหลือ )` · %เติบโตกรอกเองได้ (ค่าเริ่มต้น 15%) · ผลลัพธ์ ≤ 0 = ไม่ต้องสั่ง · ปัดขึ้นเป็นจำนวนเต็ม |
| M3 | **เดือนเป้าหมาย** | ค่าเริ่มต้น = เดือนปัจจุบันตามเครื่อง เปลี่ยนได้ (วางแผนสั่งล่วงหน้า) |
| M4 | **ตาราง Dead Stock** | เงื่อนไขทั้งสองข้อ: ขายได้ **< 60%** ของจำนวนนำเข้า **และ** ไม่มีการขายเลย **> 6 เดือน** (นับจากเดือนขายล่าสุด) |
| M5 | **การ์ดสรุป (Dashboard)** | มูลค่าสต็อกที่จม (บาท) · จำนวนรายการควรสั่ง · จำนวน Dead Stock |
| M6 | **มูลค่าสต็อกที่จม** | ผลรวม `คงเหลือ × ราคาทุน` ของสินค้าที่ Months of Supply > 6 เดือน · MoS = คงเหลือ ÷ ยอดขายเฉลี่ยต่อเดือน |
| M7 | **อันดับสินค้าขายดี** | เรียงตามจำนวนชิ้นที่ขาย (Sold) · ไม่รวมหมวด "พีโอ" · เลือกช่วงเวลาเองได้ |
| M8 | **Popup เตือน "ขายดีแต่ใกล้หมด"** | เด้งเมื่อ: อยู่ใน Top 20% ขายดี (ตามจำนวนชิ้น) และ ของเหลือขายได้ < 1 เดือน (MoS < 1) |
| M9 | **แจ้งเตือนสินค้าตามเทศกาล** | เตือนล่วงหน้า 1 เดือน ตามไฟล์ปฏิทินเทศกาล · จับคู่สินค้าด้วยคำค้นในชื่อสินค้า · รวมเปิดเทอม (เตือน เม.ย. + ต.ค.) |
| M10 | **กลุ่ม "รอคนพิจารณาเอง"** | สินค้าไม่มีประวัติขายเดือนเป้าหมายในปีก่อน ๆ → ไม่เหมาเป็น 0 แยกให้คนตัดสินใจ |
| M11 | **Export ผลลัพธ์เป็น Excel** | เอาตารางควรสั่ง / Dead Stock ไปใช้งานต่อ |
| M12 | **กรองหมวด "พีโอ" ออก** | ไม่นำมาคำนวณทุกกรณี |

## 4. ฟีเจอร์ "น่ามี" (ไว้ทีหลัง)

| # | ฟีเจอร์ |
|---|---|
| N1 | กราฟยอดขายย้อนหลัง / กราฟสัดส่วนหมวดหมู่ |
| N2 | ค่าเฉลี่ยแบบถ่วงน้ำหนัก (ปีล่าสุดน้ำหนักมากกว่าปีเก่า เช่น 50/30/20) |
| N3 | ทางเลือก: ไม่มีประวัติเดือนเป้าหมาย → ใช้ค่าเฉลี่ยทั้งปีแทน |
| N4 | ค้นหา / กรอง / จัดเรียง ในทุกตาราง |
| N5 | พิมพ์รายงานเป็น PDF / กระดาษ |
| N6 | ปรับเกณฑ์ต่าง ๆ (60%, 6 เดือน, Top 20%) ได้เองในหน้าตั้งค่า |

## 5. ข้อมูลที่ต้องเก็บ / ประมวลผล

ระบบไม่เก็บถาวร (stateless) — ตารางด้านล่างคือข้อมูลที่ระบบอ่านเข้ามาและถือไว้ระหว่างใช้งาน

### 5.1 แหล่งข้อมูลนำเข้า (ไฟล์ Excel)

| แหล่ง | ชื่อไฟล์ (ขึ้นต้น) | คอลัมน์ที่ใช้ | ให้ข้อมูล | จำนวนไฟล์ |
|---|---|---|---|---|
| ยอดขายรายเดือน | `report_by_product_YYYY_MM` | Product code, Category, Sold, Avg(cost), Avg(sell), Net profit | ยอดขายต่อเดือน, ขายดี, กำไร | หลายไฟล์ |
| สต็อกคงเหลือ | `export_product_all_...` | Product code, Category, Quantity, Cost, Price, Notify when less than | ของคงเหลือปัจจุบัน, หมวด | ไฟล์เดียว (ล่าสุด) |
| ใบนำเข้า (PO) | `export_purchase_..._YYYY_MM` | po header: create_date; line: product_code, product_quantity, product_cost | จำนวน + วันที่นำเข้า | หลายไฟล์ |
| ปฏิทินเทศกาล | (ไฟล์ผู้ใช้ทำเอง) | ชื่อเทศกาล, เดือน, คำค้นในชื่อสินค้า | เตือนเตรียมของตามเทศกาล | ไฟล์เดียว |

### 5.2 ข้อมูลที่ระบบคำนวณขึ้น (ต่อสินค้า 1 รายการ)

| ข้อมูล | ที่มา / สูตร |
|---|---|
| จำนวนนำเข้ารวม | ผลรวม product_quantity จากไฟล์ PO ทั้งหมด |
| เดือนขายล่าสุด | เดือนล่าสุดที่ Sold > 0 |
| ยอดขายเฉลี่ยเดือนเป้าหมาย (3 ปี) | เฉลี่ย Sold เดือนเดียวกัน 3 ปีย้อนหลัง |
| Months of Supply | คงเหลือ ÷ ยอดขายเฉลี่ยต่อเดือน |
| % ที่ขายได้ | ยอดขายรวม ÷ จำนวนนำเข้ารวม |
| จำนวนที่ควรสั่ง | ตามสูตร M2 |
| สถานะ | ควรสั่ง / Dead Stock / เงินจม / รอพิจารณา / ข้อมูลผิดปกติ |

### 5.3 กติกาจัดการข้อมูลผิดปกติ

| กรณี | การจัดการ |
|---|---|
| สต็อกติดลบ | ติดธง "ข้อมูลผิดปกติ" แยกไว้ตรวจ |
| สต็อกเป็นเศษ (1.5) | ปัดขึ้นเป็นจำนวนเต็ม |
| มีในยอดขาย/PO แต่ไม่มีในไฟล์สต็อก | กลุ่ม "ข้อมูลไม่ครบ ตรวจสอบเอง" |
| มีในสต็อก แต่ไม่เคยขาย/นำเข้า | เข้ากลุ่ม Dead Stock / เงินจม |
| หมวด "พีโอ" | ตัดออกทุกกรณี |
