# 🧮 ระบบคำนวณอัจฉริยะเพื่อการจัดซื้อและขจัดสินค้าค้างสต็อก

เว็บแอปไฟล์เดียวสำหรับร้านเครื่องเขียน/เครื่องใช้สำนักงาน — วิเคราะห์ไฟล์ Excel จากระบบขาย (POSPOS)
เพื่อแนะนำ **สั่งซื้ออะไร-เท่าไร** และ **หาสินค้าค้างสต็อก (Dead Stock)**

## วิธีเปิดใช้งาน
1. ดับเบิลคลิกไฟล์ **`index.html`** (เปิดด้วย Chrome / Edge)
2. ต้องต่ออินเทอร์เน็ตครั้งแรก (โหลดตัวอ่าน Excel จาก CDN)
3. ข้อมูลทั้งหมดประมวลผล **ในเครื่องของคุณเท่านั้น** ไม่ถูกส่งออกไปไหน และไม่ถูกเก็บไว้

## ขั้นตอนใช้งาน
1. **อัปโหลดไฟล์** (ลากวางได้หลายไฟล์พร้อมกัน) ระบบแยกประเภทจาก **ชื่อไฟล์**:

   | ประเภท | ชื่อไฟล์ต้องขึ้นต้นด้วย | ตัวอย่าง |
   |---|---|---|
   | ยอดขายรายเดือน | `report_by_product_YYYY_MM` | `report_by_product_2025_07.xlsx` |
   | สต็อกคงเหลือ | `export_product_all` | `export_product_all_06_07_2026.xlsx` |
   | ใบนำเข้า PO | `export_purchase` | `export_purchase_..._2025_07.xlsx` |

   > ⚠️ สำคัญ: ไฟล์ยอดขายและ PO **ต้องมีเดือน/ปี (YYYY_MM) ในชื่อไฟล์** เพราะข้างในไฟล์ไม่มีคอลัมน์วันที่
   > อัปไฟล์ยอดขายย้อนหลังหลายเดือน/หลายปีได้ (ยิ่งมี 3 ปีขึ้นไปยิ่งแม่น)

2. **ตั้งค่า**: เดือนเป้าหมาย (ค่าเริ่มต้น = เดือนปัจจุบัน) · % เผื่อการเติบโต (เริ่มต้น 15%) · ช่วงเวลาของอันดับขายดี
3. กด **▶ ประมวลผล** → ดูการ์ดสรุป + ตาราง 6 กลุ่ม + Popup แจ้งเตือน
4. กด **⬇ Export** เพื่อบันทึกผลเป็น Excel

## 📛 กติกาการตั้งชื่อไฟล์ (ละเอียด)

ระบบดู 2 อย่างจากชื่อไฟล์: (1) **ขึ้นต้นด้วยอะไร** → รู้ประเภท · (2) **มีปี-เดือน (ปปปป_ดด)** → รู้ว่าเป็นข้อมูลเดือนไหน

| ประเภทข้อมูล | ชื่อไฟล์ต้องขึ้นต้นด้วย | ต้องมีปี-เดือน | ตัวอย่างที่ถูกต้อง |
|---|---|---|---|
| ยอดขายรายเดือน | `report_by_product` | ✅ ต้องมี | `report_by_product_2025_07.xlsx` |
| สต็อกคงเหลือ | `export_product_all` | ❌ ไม่ต้อง (ใช้ไฟล์เดียวล่าสุด) | `export_product_all_2026_07.xlsx` |
| ใบนำเข้า (PO) | `export_purchase` | ✅ ต้องมี | `export_purchase_2025_07.xlsx` |

### รูปแบบปี-เดือนที่ระบบอ่านได้
- **แนะนำ:** `ปปปป_ดด` = ปี ค.ศ. 4 หลัก `_` เดือน 2 หลัก → `2025_07` = กรกฎาคม 2025
- ⚠️ **ปีต้องเป็น ค.ศ. (20xx)** ไม่ใช่ พ.ศ. → ใช้ `2025` ไม่ใช่ `2568`
- เดือน 1–12 (`07` หรือ `7` ก็ได้) · ตัวคั่นใช้ `_` `-` หรือ `.`
- รองรับรูปแบบเดิมของ POSPOS `..._วว_ดด_ปปปป` ด้วย (เช่น `06_07_2026` → อ่านเป็น ก.ค. 2026)

### ⚠️ จุดสำคัญ: วันที่ในชื่อไฟล์ POSPOS = วัน "กด export" ไม่ใช่เดือนของข้อมูล
POSPOS ตั้งชื่อไฟล์ตามวันที่กด export ระบบจึงไม่รู้เองว่าไฟล์เป็นข้อมูลเดือนไหน
**ต้องเปลี่ยนชื่อไฟล์ให้ตรงกับเดือนของข้อมูลจริงเสมอ** เช่น export ยอดขายเดือน ก.ค. 2025 → เปลี่ยนชื่อเป็น `report_by_product_2025_07.xlsx`

### ตัวอย่างชุดไฟล์ 3 ปี (วางแผนเดือนกรกฎาคม)
```
report_by_product_2023_07.xlsx     ← ยอดขาย ก.ค. 2023
report_by_product_2024_07.xlsx     ← ยอดขาย ก.ค. 2024
report_by_product_2025_07.xlsx     ← ยอดขาย ก.ค. 2025
export_product_all_2026_07.xlsx    ← สต็อกปัจจุบัน (ไฟล์เดียว)
export_purchase_2023_07.xlsx       ← PO (ย้อนหลังได้หลายไฟล์)
export_purchase_2024_07.xlsx
export_purchase_2025_07.xlsx
```

### 🛡️ ระบบกันไฟล์ทับกัน (อัตโนมัติ)
1. **อัปไฟล์เดิมซ้ำ → ถูกข้าม** (เช็กจากชื่อ + ขนาดไฟล์) พร้อมแจ้งเตือน
2. **ยอดขายเดือนเดียวกัน 2 ไฟล์ → ขึ้นเตือน "⛔ ชนกับไฟล์..."** และไม่นำมาคำนวณ ให้เอาออกไฟล์หนึ่งหรือแก้เดือน
3. **ไฟล์สต็อกใหม่ → แทนที่ไฟล์สต็อกเดิมอัตโนมัติ** (ใช้ได้ครั้งละ 1 ไฟล์)
4. **ใบ PO ที่เคยนับแล้ว → ไม่นับซ้ำ** (จำเลขที่ใบ PO ไว้) อัปไฟล์ PO ทับกันได้อย่างปลอดภัย
5. อย่ามีเลขปี 20xx อื่นปนก่อนปี-เดือนจริง — ระบบอ่านชุดตัวเลขชุดแรกที่เจอ

## 🗄️ คลังข้อมูลที่จำไว้
ระบบ **จำยอดขายรายเดือน + ประวัติการนำเข้า (PO)** ไว้ในเบราว์เซอร์อัตโนมัติ → เดือนถัดไปอัปแค่ไฟล์เดือนใหม่
- ลบเฉพาะเดือนที่ผิดได้ (กด ✕ ที่ชิปเดือน)
- **💾 สำรองเป็นไฟล์ / 📂 กู้คืน** — ใช้ย้ายเครื่อง หรือกันแคชถูกล้าง
- ⚠️ **ไฟล์สต็อกไม่ถูกจำ** ต้องอัปทุกครั้ง (ต้องใช้ยอดคงเหลือล่าสุดเสมอ)

## สูตร/เกณฑ์ที่ระบบใช้
- **จำนวนที่ควรสั่ง** = ปัดขึ้น( ค่าเฉลี่ยยอดขายเดือนเป้าหมายย้อนหลัง 3 ปี × (1 + %เติบโต) − สต็อกคงเหลือ ) · ถ้า ≤ 0 ไม่แนะนำ
- **เดือนเป้าหมาย** ค่าเริ่มต้น = **เดือนถัดไป** (เพื่อเตรียมของล่วงหน้า) เปลี่ยนได้
- **Dead Stock** = ขายได้ < 60% ของจำนวนนำเข้า **และ** ไม่ขายเลย > 6 เดือน (หรือมีของแต่ไม่เคยขาย)
- **มูลค่าสต็อกที่จม** = คงเหลือ × ราคาทุน ของสินค้าที่ Months of Supply > 6 เดือน
- **ขายดี** = เรียงตามจำนวนชิ้นที่ขาย (ไม่รวมหมวด "พีโอ")
- **%ขายออก (นำเข้า vs ขายได้)** = ขายได้รวม ÷ นำเข้ารวม · 🟢 ≥80% ขายดี · 🟠 30–80% ปานกลาง · 🔴 <30% ขายไม่ออก
- **Popup ขายดีแต่ใกล้หมด** = อยู่ใน Top 20% ขายดี และของเหลือขายได้ < 1 เดือน

## กติกาข้อมูลผิดปกติ
- สต็อกติดลบ → แยกกลุ่ม "ผิดปกติ" (ไม่คำนวณมั่ว)
- สต็อกเป็นเศษ → ปัดขึ้นเป็นจำนวนเต็ม
- หมวด **"พีโอ"** → ตัดออกทุกกรณี
- มีในยอดขาย/PO แต่ไม่มีในสต็อก → กลุ่ม "รอพิจารณา"
